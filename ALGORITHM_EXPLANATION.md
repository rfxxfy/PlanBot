# Объяснение алгоритма планирования задач PlanBot

## 📋 Общая информация

**Название алгоритма**: Deadline-Aware Hybrid Scheduling (Гибридное планирование с учётом дедлайнов)

**Назначение**: Автоматическое распределение задач пользователя по дням с учётом приоритетов, дедлайнов и индивидуальных настроек рабочего времени.

**Тип алгоритма**: Жадный алгоритм с поддержкой обратного планирования (Backward Planning) для дедлайнов.

---

## 🎯 Постановка задачи

### Входные данные:
1. **Список задач**, где каждая задача содержит:
   - `title` - название задачи
   - `hours_required` - количество часов, необходимых для выполнения
   - `priority` - приоритет (целое число от 1 до 10, где 10 - наивысший)
   - `deadline` - крайний срок выполнения (может отсутствовать)

2. **Настройки пользователя**:
   - `daily_capacity` - сколько часов в день пользователь может работать
   - `work_days` - массив рабочих дней недели (1=Пн, 7=Вс)
   - `time_zone` - часовой пояс пользователя

3. **Дата начала планирования** - обычно следующий день в часовом поясе пользователя.

---

## 🔄 Алгоритм работы

### Этап 1: Сортировка задач

Задачи сортируются для определения порядка их "посадки" в расписание:
1. **Дедлайн**: Задачи с дедлайном идут первыми.
2. **Близость дедлайна**: Чем ближе дедлайн, тем выше приоритет.
3. **Приоритет**: При равных дедлайнах (или их отсутствии) выше те, у кого больше `priority`.
4. **Длительность**: При прочих равных более короткие задачи идут первыми.

### Этап 2: Распределение по дням

Для каждой задачи выбирается стратегия в зависимости от наличия дедлайна:

#### 1. Обратное планирование (для задач с дедлайном)
Алгоритм пытается разместить задачу как можно позже, но не позже дедлайна и не раньше даты начала планирования.
- Начинаем с даты дедлайна.
- Двигаемся назад к дате начала.
- Заполняем свободные окна в рабочих днях.
- **Зачем**: Это позволяет оставить свободное место в ближайшие дни для текучки и мелких задач, гарантируя при этом, что крупная задача с дедлайном будет завершена вовремя.

#### 2. Прямое планирование (для задач без дедлайна)
Алгоритм размещает задачу в ближайшие доступные окна.
- Начинаем с даты начала планирования.
- Двигаемся вперёд.
- Заполняем свободные окна в рабочих днях.

### Этап 3: Учёт нагрузки

В каждый день алгоритм не позволяет запланировать больше часов, чем указано в `daily_capacity` пользователя. Если задача не влезает в один день, она разбивается на части и распределяется по нескольким дням.

---

## ✅ Преимущества

1. **Умное распределение**: Задачи с дедлайнами не "съедают" всё время завтрашнего дня, если их можно сделать через три дня.
2. **Гарантия сроков**: Сортировка гарантирует, что самые критичные задачи будут рассмотрены первыми.
3. **Гибкость**: Учёт рабочих дней и выходных конкретного пользователя.
4. **Timezone-aware**: Планирование и уведомления работают в локальном времени пользователя.

---

## 📊 Сложность
- **Временная**: O(N * D), где N — количество задач, D — горизонт планирования (в днях).
- **Пространственная**: O(N + D) для хранения расписания.
