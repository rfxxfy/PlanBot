# Алгоритм планирования задач

## Обзор

PlanBot использует **алгоритм планирования с учётом дедлайнов** (Deadline-Aware Scheduling Algorithm), который оптимально распределяет задачи по дням, учитывая ограничения по времени и приоритеты.

## Принцип работы

### 1. Предварительная обработка

```
Входные данные:
- Список задач (title, hours_required, priority, deadline)
- Настройки пользователя:
  - daily_capacity (часов в день)
  - work_days (рабочие дни недели)
  - time_zone (строка IANA, например Europe/Moscow)
  - work_start, work_end (рабочий интервал в формате HH:MM-HH:MM, пока используется только как метаданные для будущих слотов)
- Дата начала планирования (в таймзоне пользователя)
```

### 2. Сортировка задач

Задачи сортируются по следующим критериям (по убыванию важности):

1. **Наличие дедлайна**: Задачи с дедлайном имеют приоритет
2. **Близость дедлайна**: Чем ближе срок, тем выше в списке
3. **Приоритет**: При равных дедлайнах учитывается приоритет (1-10)
4. **Длительность**: При равных приоритетах сначала короткие задачи

```go
Пример сортировки:
1. Задача A: deadline=завтра, priority=5, hours=4
2. Задача B: deadline=послезавтра, priority=8, hours=6
3. Задача C: нет дедлайна, priority=10, hours=2
4. Задача D: нет дедлайна, priority=10, hours=5
```

### 3. Алгоритм распределения

Для каждой задачи из отсортированного списка:

#### Шаг 1: Определение временного окна

```
Если есть дедлайн:
    latest_start_date = calculate_latest_start(deadline, hours_required)
    if latest_start_date < current_date:
        return "невозможно уложиться в дедлайн"
Иначе:
    можно планировать с текущей даты
```

#### Шаг 2: Поиск свободных слотов

```
remaining_hours = task.hours_required
current_date = start_date

while remaining_hours > 0:
    if not is_work_day(current_date):
        current_date++
        continue
    
    if deadline exists and current_date > deadline:
        return "не удалось запланировать"
    
    available_hours = daily_capacity - get_scheduled_hours(current_date)
    
    if available_hours > 0:
        hours_to_allocate = min(remaining_hours, available_hours)
        
        schedule_task_for_day(task, current_date, hours_to_allocate)
        remaining_hours -= hours_to_allocate
    
    current_date++
```

#### Шаг 3: Фиксация расписания

После успешного распределения всех часов задачи, записываем в БД:
- task_schedules: связка (задача → дата → выделенные часы)
- tasks: обновляем статус на 'scheduled'

## Особенности реализации

### Разбиение задач на несколько дней

Если задача требует больше часов, чем доступно в один день:

```
Пример:
- Задача: 12 часов
- Дневная нагрузка: 8 часов

Распределение:
- День 1: 8 часов
- День 2: 4 часа
```

### Учёт рабочих дней

```go
По умолчанию: 1,2,3,4,5 (Пн-Пт)
Суббота и воскресенье пропускаются

Пользователь может настроить:
/settings 6 | 1,2,3,4,5,6  // работа в субботу
/settings 4 | 2,3,4,5      // 4 часа, Вт-Пт
```

### Обратное планирование от дедлайна

Для задач с жёстким дедлайном вычисляется последняя возможная дата начала по формуле с `ceil`:

```
deadline = 25.12.2025
hours_required = 16
daily_capacity = 8

work_days_needed = ceil(16 / 8) = 2 дня

latest_start = 25.12.2025 - 2 рабочих дня = 23.12.2025
```

Если рассчитанная `latest_start_date` оказывается **раньше** текущей стартовой даты планирования (с учетом таймзоны пользователя), задача считается невыполнимой в заданные сроки.

## Примеры работы алгоритма

### Пример 1: Простое планирование

**Входные данные:**
```
Задачи:
1. "Написать отчёт" - 4 часа, приоритет 8, дедлайн: 22.12.2025
2. "Код-ревью" - 2 часа, приоритет 5, нет дедлайна
3. "Презентация" - 6 часов, приоритет 9, дедлайн: 23.12.2025

Настройки: 8 часов/день, Пн-Пт
Текущая дата: 20.12.2025 (Пятница)
```

**Результат:**
```
20.12.2025 (Пт): 8 часов
  - "Написать отчёт" (4 часа) 📅 дедлайн завтра
  - "Код-ревью" (2 часа)
  - "Презентация" (2 часа) 📅 дедлайн 23.12

22.12.2025 (Пн): 4 часа
  - "Презентация" (4 часа) 📅 дедлайн сегодня
```

### Пример 2: Перегрузка и невозможность выполнить

**Входные данные:**
```
Задача: "Большой проект" - 40 часов, дедлайн: завтра
Настройки: 8 часов/день
```

**Результат:**
```
❌ Ошибка: Невозможно уложиться в дедлайн
Требуется: 40 часов
Доступно: 8 часов (1 день до дедлайна)
```

### Пример 3: Автоматическое разбиение

**Входные данные:**
```
Задача: "Изучить фреймворк" - 20 часов, приоритет 7, нет дедлайна
Настройки: 6 часов/день, Пн-Пт
Текущая дата: Понедельник
```

**Результат:**
```
Понедельник: 6 часов - "Изучить фреймворк"
Вторник: 6 часов - "Изучить фреймворк"
Среда: 6 часов - "Изучить фреймворк"
Четверг: 2 часа - "Изучить фреймворк"
```

## Сложность алгоритма

- **Временная сложность**: O(n × d)
  - n = количество задач
  - d = количество дней для планирования
  
- **Пространственная сложность**: O(n + d)
  - Хранение задач и слотов дней

## Преимущества алгоритма

✅ Гарантирует соблюдение дедлайнов (если это возможно)
✅ Учитывает приоритеты задач
✅ Равномерно распределяет нагрузку
✅ Автоматически разбивает большие задачи
✅ Поддерживает гибкие настройки рабочего времени
